# Sprint 0 Commit Plan

This file is an implementation guide and may be removed after the repository has been successfully initialized.

## Sprint Goal

Establish a coherent, navigable, and governed foundation for Spark OS before adding substantive mortgage or Academy content.

## Recommended First Commit

```text
Establish Spark OS foundational architecture
```

This commit should include the complete Sprint 0 package so the repository begins in a stable and understandable state.

## Why One Initial Commit

Normally, changes should be separated into small, coherent commits.

For a new empty repository, however, the folder maps, contribution rules, AI governance, and first ADR depend on one another. Committing them together creates one comprehensible starting point.

Future work should use smaller commits and branches where appropriate.

## What the Commit Establishes

- root repository purpose and navigation
- six top-level knowledge areas
- the Workbench-to-Canon lifecycle
- initial role separation
- repository-owned AI governance
- Claude's repository-steward role
- Markdown as the canonical source of truth
- contribution and review expectations

## Suggested GitHub Description

```text
The canonical knowledge operating system for Spark Mortgages.
```

## Suggested Repository Settings

After the first commit:

1. Keep the repository private during early development.
2. Set `main` as the default branch.
3. Enable Issues when the team is ready to track work outside the Markdown Parking Lot.
4. Do not require pull requests until the first foundation documents are stable.
5. Add branch protection later, when direct edits to `main` become a meaningful risk.

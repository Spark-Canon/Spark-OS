# Canon

The Canon answers:

> What has Spark OS accepted as authoritative knowledge?

The Canon contains the trusted knowledge from which training, reference material, operations, AI context, and software behavior should derive.

## Canon Standards

Canonical material should be:

- accurate
- clearly scoped
- internally consistent
- traceable to sources when facts may change
- reusable outside one course or workflow
- explicit about uncertainty and exceptions
- separated from temporary examples or exercises

## Planned Domains

- mortgage knowledge
- sales
- compliance
- operations
- software and workflow knowledge

Domain folders should be created only when real content requires them.

## Change Discipline

Canon changes should be deliberate.

When accepted Canon changes, review downstream:

- Academy lessons
- assessments
- reference material
- SOPs
- AI instructions
- Loveable content or behavior

Do not fix a derivative output while leaving its canonical source incorrect.

# Spark Brain

Spark Brain is the operating layer for the Spark OS repository.

It contains no independent business knowledge, Philosophy, Governance, Canon, Academy content, or ADR system. It loads, navigates, tracks, and validates the authoritative repository.

## Start

1. Read [`constitution.md`](constitution.md).
2. Use [`resume.md`](resume.md).
3. Select a mode in [`boot-sequence.md`](boot-sequence.md).

## Files

- [`constitution.md`](constitution.md) — operating rules and authority boundaries
- [`boot-sequence.md`](boot-sequence.md) — Quick Resume and Full Architectural Boot
- [`repository-map.md`](repository-map.md) — human-readable authority map
- [`navigation.md`](navigation.md) — task-oriented links
- [`resume.md`](resume.md) — context-recovery workflow
- [`current-state.md`](current-state.md) — current phase, priority, next task, and blockers
- [`session.md`](session.md) — last session boundary and verification state
- [`health.md`](health.md) — integrity contract and failure policy
- [`manifest.yaml`](manifest.yaml) — machine-readable boot, authority, and validation configuration

The authoritative repository architecture is defined outside Brain and linked through the repository map.
